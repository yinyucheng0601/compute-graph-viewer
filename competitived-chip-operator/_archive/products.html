Direct Baseline · NVIDIA

CUDA：成熟 GPU 全栈开发平台

CUDA 不只是 kernel 语言，而是围绕 NVIDIA GPU 构建的完整开发平台：编程模型、runtime/driver API、编译器、调试器、Profiler、数学库、框架绑定和部署生态共同构成体验优势。

核心用户GPU kernel 工程师、深度学习框架团队、HPC 工程师、推理/训练基础设施团队。
核心任务写 kernel、调试内存错误、压榨硬件性能、接入 PyTorch/TensorFlow/JAX、维护多架构二进制。
体验定位概念栈厚，但工具和资料密度高，是本报告的成熟基线。
语言/APICUDA C++、Runtime API、Driver API、NVRTC。
编译链路nvcc、PTX、ptxas、cubin，多架构 fatbin。
诊断工具Compute Sanitizer、cuda-gdb、Nsight Systems、Nsight Compute。
生产生态cuBLAS、cuDNN、NCCL、CUB/Thrust、框架默认支持。
官方 CUDA Programming Guide 13.1 已把入门内容重组为 programming model、CUDA C++、高级 CUDA、特性和技术附录五段式路径。
Nsight Compute 2026.1.1 文档强调交互式 kernel profiler、API debugger、图形化报告、baseline 比较和自定义分析规则。
CU

竞争角色

事实标准和标杆基线。对自研 SDK 来说，CUDA 是开发者默认比较对象：API 是否熟悉、工具是否足够完整、Profiler 能否解释性能、生态是否省心。

Direct Competitor · AMD

ROCm / HIP：开放 GPU 软件栈与迁移路径

ROCm 是 AMD 面向 GPU 计算的软件平台，HIP 提供 CUDA 风格的 C++ runtime API 和 kernel language。它的体验价值在于开放栈、迁移入口和供应链多元化，但版本矩阵和硬件差异会更强地影响落地。

核心用户希望支持 AMD GPU 的框架团队、平台团队、HPC/AI 基础设施团队。
核心任务从 CUDA 迁移 kernel、维护双栈、使用 rocprof/rocgdb 定位性能和正确性问题。
体验定位迁移友好但不是 drop-in replacement；需要产品化地解释 CUDA 风格 API 与 AMD 硬件模型差异。
语言/APIHIP、OpenCL、OpenMP，围绕 ROCm runtime 和驱动栈。
编译链路hipcc、Clang/LLVM AMDGPU、HSA/ROCr 运行时。
诊断工具rocprof / rocprofv3、ROCProfiler、ROCgdb、Omnitrace。
生产生态rocBLAS、MIOpen、RCCL、Composable Kernel、PyTorch ROCm。
ROCm 7.2.2 文档首页把兼容矩阵、安装、AI/HPC 使用路径和系统优化放在左侧导航，说明版本和硬件支持本身就是核心产品界面。
官方 rocprof 文档把 tracing 定义为 profiling 的第一步，用于回答哪个 kernel 最耗时、内存拷贝占比是多少等问题。
RO

竞争角色

直接替代竞品。对自研 SDK 来说，ROCm 的启发不是“照搬 CUDA”，而是如何在开放性、迁移和可诊断性之间找到平衡。

Cross-Category Benchmark · Triton

Triton：面向 DNN 自定义算子的 Python DSL

Triton 把用户从逐线程编程部分上移到 block/tile 级数据流，强调 JIT、autotune、block pointer、program_id 和 Python 工作流。它不是完整 GPU SDK 的替代品，但在自定义 DNN 算子开发效率上非常有参考价值。

核心用户AI 研究员、框架工程师、模型优化工程师、需要快速写 fused op 的 kernel 开发者。
核心任务实现 matmul、attention、softmax、layernorm、量化/反量化等 tile 友好算子。
体验定位更少底层样板代码，更快实验循环；代价是编译器和后端行为需要额外解释。
语言/APIPython、@triton.jit、triton.language、program_id、tl.load/store/dot。
编译链路Triton IR、MLIR dialects、LLVM、硬件 backend codegen。
诊断工具TRITON_INTERPRET、static/device assert、device print、第三方 profiler。
生产生态PyTorch 工作流、autotune、Gluon、Triton MLIR dialects。
Triton 官方文档把它定义为面向并行编程的语言和编译器，目标是用 Python 环境高效编写自定义 DNN compute kernel。
调试文档提供 compile-time / runtime debug operators、CPU interpreter，以及 NVIDIA/AMD 第三方工具衔接建议。
TR

竞争角色

跨界标杆。它提醒自研 SDK：用户很多时候不是想学完整硬件，而是想更快完成一个正确且足够快的算子。
