## 3 左侧面板 — 计算图

### 3.1 SVG 计算图加载

- 计算图 SVG（`input/TENSOR_LOOP_L2_bn_Unroll1_PATH3_11.svg`）以**内联方式**嵌入 HTML，避免 `file://` 协议下的 CORS 限制。
- 加载时显示旋转等待动画；加载完成后 placeholder 隐藏，SVG 插入 `#graph-transform` 容器。
- SVG 尺寸约为 **18510 × 4757 px**（原始坐标）。

### 3.2 光标和缩放

| 操作 | 行为 |
|------|------|
| 鼠标拖拽 | 平移计算图（pan） |
| 鼠标滚轮 | 以光标位置为中心缩放（scale min 5%，max 400%） |
| `Fit` 按钮 | 适配整图到视口，居中并动画过渡 |
| `+` / `−` 按钮 | 以视口中心放大 / 缩小 10% |
| 缩放比例显示 | 右上角实时百分比（`zoom-display`） |

缩放平移状态通过 CSS `transform: translate(panX px, panY px) scale(scale)` 施加到 `#graph-transform` 上。

### 3.3 节点高亮（applyStepToSVG）

每次时间轴步骤更新时，`applyStepToSVG(step)` 对 SVG 内节点应用视觉状态：

| 节点状态 | 视觉效果 |
|----------|----------|
| **Executing**（当前步算子） | 橙色高亮边框 + `executing-pulse` 脉冲动画（0.8 s 循环）|
| **Done**（已完成算子） | 绿色调低亮度 |
| **Pending**（未执行算子） | 灰色半透明 |
| **Active Tensor**（当前步 I/O） | 蓝色高亮 |
| **Live Tensor**（当前存活） | 对应层级颜色（L1蓝 / L0A紫 / UB绿 / …） |
| **Dead Tensor**（已释放） | 灰色低亮 |

### 3.4 视口自动跟随

播放或步进时，`centerOnExecuting(op, immediate)` 自动将视口居中到当前执行算子及其所有 I/O Tensor 节点的包围盒。

- **立即跳转**：手动单步时触发（`immediate=true`）。
- **平滑动画**：自动播放时触发（线性插值 factor=0.18，`requestAnimationFrame` 驱动）。

坐标计算基于 SVG `transform` 属性链 + `getBBox()`，不依赖屏幕坐标，不受 CSS 缩放影响。

### 3.5 节点点击 — 选中算子详情

点击 SVG 内任意算子节点，`attachSVGClickHandler()` 通过向上遍历 DOM 找到 `id="sprotty_operation-magic-{magic}-0"` 元素，调用 `selectOp(magic)`，右侧面板的"选中算子详情"区域随之更新。再次点击同一节点则取消选中。

### 3.6 算子快速信息栏

左侧面板底部固定一栏，随当前步骤实时刷新：

| 字段 | 含义 |
|------|------|
| Op | 算子名称（如 `TILE_A_MUL_B`） |
| Magic | 算子内部 ID |
| Inputs | 输入 Tensor magic 列表 |
| Outputs | 输出 Tensor magic 列表 |
| Tiers | 涉及的内存层级（如 `L0A, L0B, L0C`） |

### 4.3 Buffer 像素网格可视化

每个 Buffer 框内均包含一个**暗色 LCD 像素网格**（Pixel Grid），将物理字节空间映射为 2D 格子，直观展示内存占用情况。

#### 五个 Buffer 的网格规格

| Buffer | 物理容量 | 格子行 × 列 | 每格覆盖字节 | 总格子数 |
|--------|----------|-------------|-------------|---------|
| L1 | 1024 KB | 8 × 8 | 16 KB | 64 |
| L0A | 64 KB | 4 × 4 | 4 KB | 16 |
| L0B | 64 KB | 4 × 4 | 4 KB | 16 |
| L0C | 256 KB | 8 × 8 | 4 KB | 64 |
| UB | 256 KB | 8 × 8 | 4 KB | 64 |

#### 格子颜色与状态

每个 Tensor 被分配一个来自 15 色调色板（`TENSOR_PALETTE`）的固定颜色（按 magic 排序分配，保证同一 Tensor 颜色始终一致）。

| 格子视觉状态 | 触发条件 | 样式 |
|-------------|----------|------|
| **有色格子** | 该地址范围内有存活 Tensor | Tensor 对应的调色板颜色，`opacity: 0.75` |
| **Active 格子** | Tensor 是当前步的 I/O | 全亮（`opacity: 1`）+ 脉冲动画 |
| **Read 格子** | Tensor 是当前步的输入（read） | 蓝色边框（`#3b82f6`） |
| **Write 格子** | Tensor 是当前步的输出（write） | 绿色边框（`#10b981`） |
| **空格子** | 无 Tensor 覆盖 | 暗色（`#0f1829`） |

#### Tensor 地址空间布局

由于实际偏移数据不可用，各 Tensor 在 Buffer 中按**地址顺序顺序摆放**：

```
地址 0 → [Tensor A: ceil(A.bytes/bpc) 格] [Tensor B: ceil(B.bytes/bpc) 格] [空闲]
```

#### 附属信息行

每个 Buffer 框底部依次显示：

1. **Header 行**：Buffer 标签 + 占用百分比 + 已用/总容量（如 `36.0 KB / 1024 KB`）
2. **Sub-info 行**：容量、访问延迟、数据格式简介（如 `1024 KB · ~20cy · row-major`）
3. **Free 行**：剩余空间提示（如 `··· 988 KB FREE ···`）
4. **地址行**：起始/结束十六进制地址（如 `0x000000 → 0x0FFFFF`）
5. **Legend 行**：最多显示 5 个 Tensor 的色块 + 名称；超出部分显示 `+N…`

## 6 数据模型与核心算法

### 6.1 OP_DATA — 算子静态数据

```javascript
{ m: 10079, n: 'TILE_A_MUL_B', i: [46, 47], o: [126] }
```

| 字段 | 含义 |
|------|------|
| `m` | 算子 magic ID（全局唯一整数）|
| `n` | 算子名称 |
| `i` | 输入 Tensor magic 列表 |
| `o` | 输出 Tensor magic 列表 |

共 **166 条**算子记录，涵盖 7 种操作类型：

| 算子类型 | 含义 |
|----------|------|
| `TILE_COPY_IN` | DDR → L1 或 L1 → UB DMA 搬移 |
| `TILE_COPY_OUT` | L1/UB → DDR 或层间 DMA 写回 |
| `TILE_L1_TO_L0A` | L1 → L0A 数据搬移（Cube A 输入）|
| `TILE_L1_TO_L0B` | L1 → L0B 数据搬移（Cube B 输入）|
| `TILE_A_MUL_B` | Cube 矩阵乘（L0A × L0B → L0C）|
| `TILE_A_MULACC_B` | Cube 矩阵乘累加（L0C 结果累加）|
| `TILE_MULS/EXP/SUB/…` | Vector 单元算子（UB 内运算）|

### 6.2 TENSOR_META — Tensor 元数据

```javascript
55: { sh: [32, 64], dt: "BF16", b: 4096, s: "a_l0" }
```

| 字段 | 含义 |
|------|------|
| `sh` | 张量形状（维度数组）|
| `dt` | 数据类型（`BF16`、`UINT32`、`FP32` 等）|
| `b` | 字节大小（`sh` 各维度乘积 × 每元素字节数）|
| `s` | 符号名（来自源代码变量名，如 `kRope`、`a_l0`）|

共 **159 条** Tensor 记录，数据从实际 JSON 编译产物中提取。

### 6.3 拓扑排序调度

使用 **Kahn 算法**（BFS 拓扑排序）对 166 个算子生成确定性执行顺序：

1. 建立 `tensorConsumers` 映射：每个 Tensor 被哪些算子消费。
2. 初始化入度（`inDeg`）：算子的未满足输入依赖数。
3. 将所有预存在 Tensor（`PRE_EXISTING`，即 DDR 输入）标记为"已就绪"。
4. 将初始入度为 0 的算子加入队列，每轮排序后取出（`sort` 保证确定性），标记输出 Tensor 为就绪，更新下游算子入度。
5. 若存在循环依赖或孤立算子，追加到调度末尾。

结果存入 `SCHEDULE`（166 个 magic ID 的有序数组）。

### 6.4 Tensor 生命期计算

```
born[t] = scheduleIndex(producer_op)        // Tensor 产生时刻
dies[t] = max(scheduleIndex(consumer_op))   // 最后被消费时刻
```

- **预存在 Tensor**（DDR 输入）：`born = 0`，`dies = SCHEDULE.length - 1`（全程存活）。
- **无消费者 Tensor**：`dies = born`（仅存活一步）。

`getLiveTensorsAtStep(step)` 返回满足 `born <= step <= dies` 的所有 Tensor，按层级分组，每层内按 magic 排序。

### 6.5 Tensor 内存层级推断

`getTensorTier(magic)` 根据**产生该 Tensor 的算子名**推断其所在内存：

| 产生算子 | 推断层级 |
|----------|----------|
| 明确在 `DDR_TENSORS` 集合中 | DDR |
| `TILE_L1_TO_L0A` | L0A |
| `TILE_L1_TO_L0B` | L0B |
| `TILE_A_MUL_B` / `TILE_A_MULACC_B` | L0C |
| 任意 UB 向量算子 | UB |
| `TILE_COPY_IN`（输入来自 DDR）| L1 |
| `TILE_COPY_IN`（输入来自 L1）| UB |
| `TILE_COPY_OUT`（输出到 DDR）| DDR |
| 无产生算子 | DDR |
