芯片/算子开发者旅程地图

相比普通开发工具，GPU/算子产品的旅程低谷更集中在“环境是否可用、错误是否可解释、性能是否可复现”。这三类焦虑决定 P0 优先级。

旅程风险与产品机会

风险典型症状竞品参照产品机会
兼容性风险安装成功但运行失败，custom op ABI 需要重建。ROCm 兼容矩阵、CUDA 容器生态。Stack Doctor + 修复命令。
正确性风险非法访问、异步错误、数值误差定位慢。Compute Sanitizer、Triton interpreter。Error Explainer + reference diff。
性能风险指标多但不可行动，实验不可复现。Nsight source correlation / baseline。Profile Notebook。
生产风险本地可跑，CI 或升级后回归。CUDA 默认生态、ROCm 镜像策略。Production Gate。
