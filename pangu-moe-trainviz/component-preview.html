训练指标折线图 · training-metrics-chart

为 TrainScope（盘古训练透视）的「模型效果时间轴①」与「参数信号面板④」提供一个可复用的数值时序折线图：多 series、异常尖峰高亮、框选兴趣窗口、step 游标。设计系统现有 pattern 无一渲染数值曲线，故按审批门提案为新共享 pattern。

⛔ 这是预览，未进业务页、未吸收进子模块。请你看过后决定是否批准；批准后我再抽成 patterns/training-metrics-chart/{pattern.html,css,js,json} 并注册。

为什么要新建 缺口论证

现有最接近：swimlane-task（canvas 甘特任务条，按 lane × 时间画分段事件）。它表达「区间型事件」，无法表达连续数值曲线（loss/grad/eval）、双轴、异常点、框选区间。其余 pattern 是图/IDE 外壳/播放条，均非时序曲线。
  • 复用价值：loss / val / eval / grad norm / load-balance / MFU / 任意 per-step 指标都能复用，一次投入长期收益。
  • 契合 PTO 规则：曲线用 viz 色带（--highlight-*），异常用语义 P0 红，框选用 primary 选择态——viz 色与 UI 色分离，至多一层可见边界，无阴影。

提案 API & token 用法 契约

  • PtoTrainingMetricsChart.render(container, spec) → controller
  • spec = { steps[], series[{id,label,key,colorVar,axis}], data{key:[]}, anomalies[{step,seriesId}], interestWindow, cursor, onBrush, onCursor }
  • controller = { setInterestWindow(win), setCursor(step), setData(data), setAnomalyVisible(bool), destroy() }
  • series 配色仅取 --highlight-*-source（蓝/紫/绿/橙/深紫）；异常 = --danger；框选 = --primary；轴/网格 = --border-*、文字 = --foreground-muted。
  • forbidden：series 用语义状态色、坐标轴加阴影、卡中卡、把异常高亮也画成 viz 色。
