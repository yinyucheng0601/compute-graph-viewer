优势控制力最强，适合极限优化和复杂硬件特性。迁移路径清楚，适合维护 CUDA/HIP 双栈。样板少、实验快、autotune 贴合 Python 工作流。
劣势开发周期长，正确性和性能都靠专家经验。迁移后仍需重调，性能差异常来自硬件心智不一致。不适合所有控制场景，后端行为和边界限制需要解释。
最佳任务自定义 memory layout、底层同步、硬件特性。已有 CUDA 资产迁移和供应链多元化。matmul、softmax、layernorm、attention、量化/反量化。
总结专家模式。迁移模式。生产率模式。
