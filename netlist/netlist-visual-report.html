网表到图算法的抽象

本页建立后续讨论的基础模型：在芯片物理设计中，网表不是静态连接清单，而是可被算法处理的结构化图。图中将 instance、cell、macro 映射为节点，将 net、bus、clock 映射为超边，并把时序、流量、拥塞、功耗等信息转化为权重和属性。该抽象参考了开源 P&R 工具链中常见的 netlist-to-graph 建模方式，用于解释 placement、routing 和拥塞预测的输入结构。

元素定义

Nodeinstance、cell、macro、register 等对象。
Hyperedge一个 net 连接多个 pins / cells 时形成的超边。
Edge Weight由 timing、traffic、congestion、power 共同决定的综合代价。
Node Attributearea、clock domain、power domain、fixed、halo、affinity 等属性。
核心映射
Instance / Cell / Macro -> Node Net / Bus / Clock -> Hyperedge Timing / Traffic -> Edge Weight Power / Clock Domain -> Node Attribute

CANN / Ascend Traffic-aware Placement

本页描述软件栈信息如何进入物理设计约束。CANN Graph Engine、算子库、通信库和 profiling 数据可以形成 workload semantics；这些语义不替代硬件网表，而是补充为 traffic matrix、net weight、placement affinity 和 routing priority。图表的重点是把 AI workload 中的高频通信和高带宽数据流前移到布局阶段，避免仅以线长或局部时序作为优化目标。

白皮书解读

1软件执行图、算子融合和集合通信模式形成 workload semantics。
2Profiling 将带宽、复用频率和热点路径量化为 traffic matrix。
3Traffic matrix 叠加到 netlist，形成可驱动布局的 weighted nets。
4Placement/routing 产生 timing、congestion、IR/EM 等 QoR，再回写约束。
关键点：CANN profiling 不替代硬件 netlist，而是为 netlist 增加 workload-aware 权重。

Ascend / CANN 约束映射

本页将公开资料中的 CANN 软件栈能力映射到芯片物理设计语言。由于公开资料没有披露 Ascend 910B/950 的真实版图级网表、PDK 或 floorplan，图表采用方法论表达：从 Graph Engine、Ascend C、通信库和 runtime 出发，抽取 workload semantics，再沉淀为 netlist constraints，最终进入 floorplan、placement、CTS、routing 和 signoff 的闭环。

约束层级

层级作用
CANN StackGraph Engine、Ascend C、Operator Library、Communication Library。
Workload Semantics算子融合、traffic matrix、memory locality、AllReduce pattern。
Netlist Constraintsnet weight、带宽类别、布局区域、布线优先级、电源/时钟域。
P&Rfloorplan、placement、CTS、routing、check，并把 QoR 反馈回来。

边界：公开资料不包含 Ascend 910B/950 的真实版图级 netlist、PDK 或 floorplan；图表达的是设计约束映射方法。

真实 Yosys + netlistsvg 生成图

本页展示真实工具链输出：先用 Verilog 描述一个 Ascend tile control path，再通过 Yosys 综合为 JSON 网表，最后由 netlistsvg 自动绘制成数字逻辑网表图。该图包含 mux、comparator、adder、xor、async reset DFF、bus slice 和常量等标准结构，证明“可综合 RTL -> JSON netlist -> schematic visualization”的链路已经跑通。

生成流程

commands
yosys -p 'read_verilog ascend_tile_control.v; proc; opt; memory; opt; write_json ascend_tile_control_yosys.json' ./node_modules/.bin/netlistsvg ascend_tile_control_yosys.json -o images/ascend-tile-control-netlistsvg.svg
Verilog
ascend_tile_control.v
Yosys JSON
ascend_tile_control_yosys.json
