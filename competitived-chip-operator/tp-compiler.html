Touchpoint 05 · Compiler

编译器体验的关键是解释“我的代码被改成了什么”

算子平台越自动化，开发者越需要知道 JIT、IR、target arch、寄存器、occupancy 和 fallback 的影响。编译器不能只在失败时出现，它应成为性能解释链的一部分。

优势PTX/SASS、编译选项和 profiler 指标之间的生态成熟，专家可下钻。LLVM 开放性强，适合接入自研工具和 compiler pass 解释。JIT 与 Python 工作流融合，自动生成和 autotune 的迭代速度快。
劣势底层闭源部分限制完全解释；新手不易理解 PTX/SASS 与源码的关系。错误信息常暴露底层栈，迁移用户需要知道更多工具链组件。自动优化边界不透明，IR/backend 指标和源码之间缺默认可视化。
失败体验常见是 arch mismatch、模板错误、register pressure，需要更好的归因。常见是 gfx 支持、依赖版本、clang 错误，信息量大但导航弱。常见是 unsupported op、shape/meta 参数、JIT cache，Python 用户容易被 compiler 细节打断。
总结专家可解释性最强。开放但需要包装。高效但要补透明度。
